ZigZag Conversion

Question

The string “PAYPALISHIRING” is written in a zigzag pattern on a given number of rows like this: (you may want to display this pattern in a fixed font for better legibility)

P   A   H   N
A P L S I I G
Y   I   R

And then read line by line: “PAHNAPLSIIGYIR”
Write the code that will take a string and make this conversion given a number of rows:

string convert(string text, int nRows);

convert(“PAYPALISHIRING”, 3) should return “PAHNAPLSIIGYIR”.

Answer

先分成n行，再合并

这个方法很容易想到，因为比较直观嘛。先把给定的字符串按照zigzag转换的方式排序好，分别按行放入n个字符串，然后再将这n个字符串串联起来，得到结果。
代码如下：

// create by coffer
string convert(string s, int numRows) {
    string *str_array;
    int i;
    int str_length = s.length();

    if (str_length <= 1 || numRows == 1) { // 排除特殊情况
        return s;
    }

    // 申请并初始化这n个字符串,用于存放zigzag转换
    str_array = new string[numRows]; 
    for (i = 0; i < numRows; i++) {
        str_array[i].clear();
    }

    int mod_num = numRows * 2 - 2;
    int index;
    for (i = 0; i < str_length; i++) {
        index = i%mod_num;
        if (index >= numRows) {
            // index = (numRows - 1) - (index - numRows +1)
            index = 2 * numRows - index - 2; 
        }
        str_array[index] += s[i];
    }

    string result;
    result.clear();
    for (i = 0; i < numRows; i++) {
        //cout << str_array[i] << endl;
        result += str_array[i];
    }
    return result;
}

上面算法的改进

主要想法是直接从源字符串中按照顺序挑出要放到结果里的字符，理应可以简化很多中间步骤，这里需要推导一下公式。
对于4行的数，（4*2-2=）6个数是一个循环

1    1    1
2  6 2  6 2
3 5  3 5  3 ……
4    4    4

因此，将行数numRows*2-2这一公式的计算结果记为mod。则，第一行第一个数的下标为mod*0+0,第一行第二个数的下标为mod*1+0……
第二行第一个数的下标为mod*0+1,第二行第二个数的下标为mod*0+(mod-1),第二行第三个数的下标为mod*1+1,第二行第四个数的下标为mod*1+(mod-1)
依次类推可得每一行每一个字符在源字符串中的位置。
因此确立公式，直接从源字符串转换到目标字符串，省去了中间的步骤。
代码如下：

// create by coffer
string convert(string s, int numRows) {
    int mod_num;
    int row, i;
    int str_length = s.length();
    string result_str;
    int next_index, next_index_2;

    if (str_length <= 1 || numRows <= 1) {
        return s;
    }

    mod_num = 2 * numRows - 2;
    //result_str.clear();
    row = 0;

    // 首行
    next_index = 0;
    while (next_index < str_length) {
        result_str += s[next_index];
        next_index += mod_num;
    }

    // 中间行
    for (i = 1; i < numRows - 1; i++) {
        next_index = i;
        next_index_2 = mod_num - i;
        while (next_index_2 < str_length) {
            result_str = result_str + s[next_index] + s[next_index_2];
            next_index += mod_num;
            next_index_2 += mod_num;
        }
        if (next_index < str_length) {
            result_str += s[next_index];
        }
    }

    // 末行
    next_index = mod_num / 2;
    while (next_index < str_length) {
        result_str += s[next_index];
        next_index += mod_num;
    }

    return result_str;
}

但是意料之外的是，结果还是一样，才击败20%，可能O(n)确实还不够快，需要新的算法。

很精巧的代码

引用自桑海的博客：http://www.cnblogs.com/sanghai/p/3632528.html

string convert(string s, int nRows){
    if(nRows == 1) return s;
    string res[nRows];
    int i = 0, j, gap = nRows-2;
    while(i < s.size()){
        for(j = 0; i < s.size() && j < nRows; ++j) res[j] += s[i++];
        for(j = gap; i < s.size() && j > 0; --j) res[j] += s[i++];
    }
    string str = "";
    for(i = 0; i < nRows; ++i)
        str += res[i];
    return str;
}

好简洁！不过复杂度还是在O(n)，等以后找到了更好的代码再贴上来吧~
