Android学习之"Android Studio安装总结"

从开始安装AS到可以跑起程序来我总共花了5天，是我太水还是太水了呢-。-

从下载安装AS，到我跑起来第一个程序，前后共5天，说来真是惭愧啊。当然也不是说5天全在搞这个，刚开始其实积极性不大高，沉迷于怪物猎人呢。下面总结一下我遇到的问题~

Failed to install Intel HAXM报错

最开始遇到的问题是安装到android的模拟器加速器（intel HAXM）这一步时，报错。

原因是VT-x没有开启，即BIOS里面的“Intel Virtualization Technology”选项没有打开，在Bios中修改。如下图：


参考教程：安装android studio报错Failed to install Intel HAXM

Unfortunately报错

其实我最开始没管上面这个问题，装好AS后试过，程序编译运行会unfortunately报错，如下图：

那就先把问题1解决，然后再来看这一问题。在开始我以为是虚拟设备版本太高，就新建了一个Android4.0的虚拟设备，并没有解决。然后觉得是SDK不支持，于是把SDK全下了下来（想想都不可能- -），还是不行，接着把AS卸了重装，仍然不行。

好吧，自己想到的方法都尝试过了，开始查找资料，在AS官方论坛发现有好些内容，不过最终还是这篇资料帮助到了我：Android Studio开发环境部署

这才发现原来先要安装和配置JDK和jre，虽然我之前有开发过JAVA，但是电脑是否重装过，配置是否还有效等等都已经不清楚了啦。

JDK环境变量配置

安装好JDK（我的电脑是装64位的）和jre后，配置环境变量，

新建三个系统变量：

JAVA_HOME   C:\Program Files\Java\jdk1.7.0_51
JRE_HOME    C:\Program Files\Java\jre7
CLASSPATH   .%JAVA_HOME%\lib;%JAVA_HOME%\lib\tools.jar

编辑系统变量–Path变量，注意不是新建，是填入：

%JAVA_HOME%\bin;%JAVA_HOME%\jre\bin;

如果原本有内容的话要用”;”隔开.

参考教程：JDK环境变量配置–win8版

重新安装AS

配置好JDK后，重新安装Android Studio。
下载地址：Android Studio中文社区

安装好后，直接创建一个包含有activity的工程，就直接能在虚拟设备上运行了。
可算是搞定了。
